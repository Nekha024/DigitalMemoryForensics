System FAQ

We use a strict **Retrieval-Augmented Generation (RAG)** framework. The AI is architecturally restricted to only generate responses based on the provided evidence blocks. If a data point is not present in the ingested evidence, the system provides a "null" status rather than inferring information.
MEMOIR integrates with standard forensic imaging protocols to ingest recovered deleted items from unallocated space. For encrypted data, once the investigator provides the decryption key or certificate, the engine indexes the contents into the secure vector database for semantic analysis.
The system is designed for horizontal scalability. Utilizing a distributed vector database architecture, DMF can process and index petabytes of data across multiple nodes, maintaining sub-second query latency even as the investigative corpus grows.
